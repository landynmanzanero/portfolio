Overview

Commonwealth Bank (CommBank) is Australia's largest financial institution, serving 17M+ customers across transaction banking services.

The Payments Team's product, PaaS (Payments-as-a-Service), is a suite of APIs that makes connecting directly with CommBank's payment infrastructure seamless and cost-effective for corporate clients.

CommBank's goal is to build a developer-first PaaS platform to drive adoption in an increasingly competitive landscape. Thus, we partnered with the Payments team to explore:

How might we‚Äîcorporate payments‚Äîlearn from developer-first platforms like Twilio, Stripe, and Snowflake?

Research

Competitive Analysis

  • We benchmarked five leading API providers to identify what defines best-in-class developer experience (DevEx)
  • Examined tools, features, and services that address common user concerns

Competitors analyzed: Adyen, Plaid, Twilio, Stripe, and others

User Interviews

  • Spoke with 10+ engineers and developers from companies like Google, Oracle, and IBM
  • Identified key factors that drive platform adoption: what developers value in third-party systems, what builds trust, and where onboarding friction occurs

Initial Findings

1. Documentation: Poor documentation leads to unnecessary trial-and-error during onboarding

2. Integration: APIs that plug in and test quickly are preferred; issues arise when failure points are unclear

3. Security: Strong security builds trust, though required steps can slow development

4. Customer Support: Fast, reliable responses significantly improve DevEx

User Personas

We identified two primary user personas based on our research:

Developer Dara - The Implementer

28-year-old software developer at a medium-sized tech company. Responsible for integrating third-party APIs into the team's product. Values documentation that comes first with reliable tests and SDKs.

"Documentation comes first. It's the most straightforward way to understand how an API works."

Engineer Eli - The Adopter

45-year-old senior software engineer responsible for maintaining the payment system. Evaluates third-party APIs for integration and hands off implementation tasks. Values trust and reliability above all.

"Trust is essential ‚Äî if you can't rely on it, you won't build on it."
